## 7. Knowledge architecture

Knowledge should be classified by function.

| Knowledge area | Classification | Capability supported | Reason included | Notes |
|---|---|---|---|---|
|  | `foundational | enabling | contextual | optional | replaceable | inherited_but_justified | inherited_weakly_justified` |  |  |  |

### 7.1 Knowledge classification meanings

| Classification | Meaning |
|---|---|
| Foundational | Required for many later capabilities or domains. |
| Enabling | Needed to perform a specific capability well. |
| Contextual | Helps learners understand a situation, system or problem. |
| Optional | Valuable but not required for all learners. |
| Replaceable | One of several possible knowledge routes. |
| Inherited but justified | Traditional curriculum content retained after derivation. |
| Inherited weakly justified | Traditional content included only provisionally or flagged for review. |
