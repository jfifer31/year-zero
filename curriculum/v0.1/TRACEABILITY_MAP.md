# Traceability Map – Version 0.1

This file maps curriculum elements back to the research chain that produced them.

It is one of the most important integrity files in the project.

If a curriculum element cannot be traced back to prior research, it should be marked as provisional, revised or removed.

---

## 1. Purpose

The traceability map answers:

> Why is this curriculum element here?

It links:

```text
source / evidence
→ claim
→ stage finding
→ pass-forward statement
→ capability / function
→ curriculum element
→ evidence of learning
```

---

## 2. Status

**Status:** `not_started`

**Primary authorising stages:** Stages 10–11, using authorised records from Stages 1–9

**Maintained by:** Claim Steward / Curriculum Architect / Release Manager

---

## 3. Traceability rules

Every major curriculum element should have:

- curriculum element ID;
- originating stage or stages;
- supporting claim IDs;
- related human invariant or enduring function;
- related capability;
- evidence confidence;
- assumptions;
- limitations;
- evidence-of-learning route.
